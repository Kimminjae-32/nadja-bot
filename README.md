# 🎮 나쟈 봇 (Nadja Bot)

이터널 리턴 내전/구인 전용 디스코드 봇입니다.

---

## ✨ 주요 기능

### ⚔️ 구인
- 일반/랭크 게임 구인 메시지 생성
- 참가/취소 버튼으로 간편 참가
- 인원 달성 시 방장에게 DM 알림
- 구인 메시지 자동 삭제 (기본 24시간)

### 🏟️ 내전

| 커맨드 | 설명 |
|---|---|
| `/내전 루미아 섬` | 팀당 인원/최대인원 자유 설정 |
| `/내전 론울프` | 1인 1팀 개인전 (최대 18명), 포지션 불필요 |
| `/내전 코발트` | 4vs4 고정 |

---

## 📋 내전 참가 방식

모든 내전 유형은 **웹 폼**으로 참가합니다.

1. 디스코드 메시지의 **참가/취소** 버튼 클릭
2. 개인 웹 링크로 이동 → **디스코드 닉네임 / 인게임 닉네임 / 선호 포지션** 입력 후 신청
   - 론울프는 포지션 입력 없음
3. 신청 완료 후 **수정 / 취소 페이지** 개별 제공

### 디스코드 버튼

| 버튼 | 설명 | 권한 |
|---|---|---|
| 참가/취소 | 웹 폼 링크 제공 (이미 신청 시 즉시 취소) | 누구나 |
| ⚙️ 관리 | 관리자 웹 페이지 링크 제공 | 방장 |

---

## ⚙️ 관리자 웹 페이지

방장만 접근 가능한 웹 기반 관리 패널입니다.  
참가 신청 시 표시되는 배너 또는 `⚙️ 관리` 버튼으로 접근합니다.

### 기본 기능

| 기능 | 설명 |
|---|---|
| 전체 목록 | 참가자 목록 조회 / 팀 직접 배정 / 개별 삭제 |
| 팀별 보기 | 팀 카드 형태로 배정 현황 확인 |
| 🎲 자동 배정 | 참가자를 팀 수에 맞게 랜덤 배정 |
| 🃏 실험체 랜덤 배정 | 팀 배정된 참가자에게 실험체 무작위 지정 |
| 📤 디스코드에 결과 전송 | 팀 배정 결과를 채널에 임베드로 전송 |
| 🗑️ 모집 종료 | 디스코드 메시지 삭제 및 데이터 정리 |

참가자 목록은 **5초마다 자동 갱신**됩니다.

### 🚫 캐릭터 밴 (내전 전용)

- 전체 실험체 목록을 그리드로 표시
- 클릭으로 밴/언밴 토글
- 밴된 실험체는 **실험체 랜덤 배정** 풀에서 제외

### 🎯 팀경매(드래프트) (내전 전용)

팀장이 순서대로 참가자를 직접 선택하는 드래프트 방식입니다.

**진행 순서**
1. 관리자 페이지에서 팀별 팀장 선택
2. **드래프트 시작** → 팀장마다 개인 픽 URL 생성
3. 각 팀장에게 URL 전달 → 라운드로빈 순서로 참가자 픽
4. 관리자 페이지에서 실시간 픽 진행 상황 모니터링

**팀장 픽 페이지**
- 내 차례일 때만 참가자 카드 클릭 가능
- 3초마다 자동 갱신

## 🗺️ 내전 유형별 상세

### 루미아 섬
```
/내전 유형:루미아 섬 팀당인원:3 최대인원:24
→ 8팀 (3인팀 x8), 최대 24명
```

### 론울프 (개인전)
```
/내전 유형:론울프 최대인원:18
→ 1인 1팀, 최대 18명
→ 포지션 입력 없음 (인게임 닉네임만)
→ 실험체 랜덤 배정 지원
```

### 코발트
```
/내전 유형:코발트
→ 4vs4 고정 (8명)
```

## 🌐 웹 페이지 구성

| 경로 | 설명 |
|---|---|
| `/join?event=ID&discord_id=ID` | 참가 신청 폼 |
| `/cancel?token=TOKEN` | 참가 취소/수정 페이지 |
| `/admin?event=ID&token=TOKEN` | 관리자 페이지 (방장 전용) |
| `/draft/:eventId/:captainToken` | 드래프트 팀장 픽 페이지 |

## 🛠️ 사용 기술

- **Node.js**
- **discord.js** v14
- **express** (웹 폼 서버)
- **node-cron** (자동 삭제 스케줄러)
- **JSON 파일 DB** (`nadja-events.json`)
- **Eternal Return Open API**

## 📝 참고사항

- 구인 메시지는 기본 24시간 후 자동 삭제됩니다
- 방장도 일반 참가자와 동일하게 웹 폼으로 참가 신청 가능
- 여러 서버에서 동시 사용 시 각 서버의 내전 데이터가 독립적으로 관리됩니다
- 웹 폼 참가자가 이미 신청한 상태에서 `참가/취소` 버튼 클릭 시 즉시 취소됩니다
